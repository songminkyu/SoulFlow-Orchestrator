# Phase Gates

작업 단계 전환 시 충족해야 할 체크리스트. LLM이 평가.

## PLAN → IMPL

- [ ] 스펙 또는 작업 명세 존재
- [ ] 수정 대상 파일 목록 확정
- [ ] 리스크/의존성 기록

## IMPL → REVIEW

- [ ] 스펙 기준 파일 수정 완료
- [ ] 셀프 검증 통과 (빌드, 기본 테스트)
- [ ] 변경 범위가 스펙과 일치

## REVIEW → VALIDATE

- [ ] CRITICAL 이슈 0건
- [ ] HIGH 이슈 해결 또는 사용자 승인
- [ ] 코드 스타일/컨벤션 준수

## VALIDATE → COMPLETE

- [ ] 빌드 성공
- [ ] 테스트 통과
- [ ] lint 통과

## 에스컬레이션

Gate 미충족 시:
1. 미충족 항목 명시
2. 해결 방안 제시
3. 사용자 판단 요청 (강제 통과 가능)
