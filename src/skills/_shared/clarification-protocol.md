# Clarification Protocol

모호한 요청에 대한 불확실성 분류 기준.

## 분류

| 수준 | 판단 기준 | 행동 |
|------|----------|------|
| LOW | 기본값 적용 가능, 한 가지 해석만 합리적 | 진행. 가정을 명시. |
| MEDIUM | 2-3개 합리적 해석 존재 | 선택지 제시 후 확인. |
| HIGH | 비즈니스/보안 결정, 되돌리기 어려운 변경 | 차단. 사용자 확인 필수. |

## HIGH 트리거 (반드시 확인)

- 비즈니스 로직 결정 (가격 정책, 권한 모델)
- 보안 관련 선택 (인증 방식, 암호화 범위)
- 파괴적 작업 (데이터 삭제, 스키마 변경)
- 주관적 표현 ("더 좋게", "깔끔하게", "최적화")

## MEDIUM 처리

```
"이 작업에 대해 몇 가지 접근이 가능합니다:
1. [옵션 A] — [장점/단점]
2. [옵션 B] — [장점/단점]
어떤 방향이 맞을까요?"
```

## LOW 처리

```
"[가정]을 전제로 진행합니다. 다른 의도였다면 알려주세요."
```
