# Error Escalation

에러 발생 시 대응 규칙.

## 규칙

| 상황 | 행동 |
|------|------|
| 동일 에러 3회 반복 | 전략 변경. 다른 접근 시도. |
| 범위 외 파일 접근 | 차단 + PL에 보고 |
| 미지원 기능 발견 | 사용자 확인 요청 |
| 의존성 충돌 | 구체적 에러 + 해결 방안 보고 |
| 빌드/테스트 실패 | 에러 메시지 + 관련 코드 함께 보고 |

## 보고 형식

```
**차단됨**: [무엇이 막혔는지]
**시도한 것**: [시도한 접근들]
**증거**: [에러 메시지/로그]
**제안**: [다음 단계 옵션]
```

## 금지 사항

- 동일 명령 반복 실행 (3회 제한)
- 에러 무시하고 진행
- 범위 밖 파일 임의 수정
