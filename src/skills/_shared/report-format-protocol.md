# 구조화된 보고서 형식 프로토콜

## 3요소 필수 구조

모든 에이전트 보고서는 다음 3요소를 반드시 포함해야 합니다:

### 1. 발견 (What)
- 무엇을 발견했는지 — 현상, 문제, 완료된 작업
- 구체적 수치/파일명/오류 메시지 포함
- 예시: "src/auth.ts 파일에서 SQL Injection 취약점 3건 발견"

### 2. 조치 (Action)
- 무엇을 수정/완료했는지 — 변경 사항 구체 기술
- 파일명, 라인 번호, 변경 내용 포함
- 예시: "parameterized query로 교체 (auth.ts:42, 67, 89)"

### 3. 판단 근거 (Why)
- 왜 그렇게 판단했는지 — 근거/기준/맥락
- 관련 규칙, 베스트 프랙티스, 비즈니스 요구사항 참조
- 예시: "OWASP Top 10 A03:2021 기준 + 내부 보안 가이드라인 준수"

## 포맷 템플릿

```markdown
## 보고

**발견**: [현상/문제/완료 작업]

**조치**: [변경 사항]

**판단 근거**: [근거/기준]
```

## Validator 분기 로직

```
오류 자동 체크 (빌드/테스트/lint)
    ↓
┌─ 오류 수 ≤ 5 → AI 즉시 수정 → 재검증 → done
└─ 오류 수 > 5 → Implementer에게 반환
                  (오류 목록 + 수정 방향 포함 보고서)
```
