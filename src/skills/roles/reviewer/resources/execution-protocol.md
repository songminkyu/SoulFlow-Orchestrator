# Reviewer Execution Protocol

## 1. 변경 파악

- 변경 파일 목록 확인
- diff 분석
- 스펙 대비 변경 범위 확인

## 2. 코드 검토

- 정확성: 로직이 스펙과 일치하는가
- 보안: 주입/XSS/인증 취약점
- 성능: 불필요한 연산, 메모리 누수
- 컨벤션: 기존 코드 스타일 준수
- 복잡도: YAGNI/DRY/KISS 위반

## 3. 이슈 분류

| 심각도 | 정의 | 행동 |
|--------|------|------|
| CRITICAL | 보안/데이터 손실 위험 | 즉시 수정 필수 |
| HIGH | 기능 오류 | 수정 권장 |
| MEDIUM | 개선 사항 | 선택적 수정 |
| LOW | 스타일/가독성 | 참고 |
