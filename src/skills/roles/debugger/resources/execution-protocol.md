# Debugger Execution Protocol

## 1. 증상 파악

- 에러 메시지/로그 확인
- 재현 조건 파악
- 영향 범위 추정

## 2. 범위 축소

- 관련 파일 탐색
- 호출 경로 추적
- 이분 탐색으로 원인 위치 특정

## 3. 근본 원인 분석 (RCA)

- 직접 원인 vs 근본 원인 구분
- 코드 흐름 추적
- 데이터 상태 확인

## 4. 보고

```
**증상**: [에러 메시지/동작]
**재현**: [재현 조건]
**원인**: [근본 원인]
**수정 제안**: [구체적 코드 변경]
**영향**: [수정 시 영향 범위]
```

## 금지 사항

- 원인 미확인 상태에서 코드 수정
- 증상만 보고 추측 수정
