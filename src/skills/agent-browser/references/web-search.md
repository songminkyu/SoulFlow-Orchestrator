# web_search 레퍼런스

## 기본 사용

```
web_search(query="검색어", max_results=5)
```

## 쿼리 전략

| 상황 | 전략 | 예시 |
|------|------|------|
| 최신 정보 | 연도/날짜 명시 | `"python 3.12 2024"` |
| 특정 사이트 | site: 연산자 | `"site:github.com openai sdk"` |
| 정확 구문 | 따옴표 | `"exact phrase"` |
| 제외 | 마이너스 | `"react hooks -class"` |
| 파일 형식 | filetype: | `"filetype:pdf 보고서"` |

## 쿼리 설계 원칙

- **좁게 시작** — 넓은 쿼리는 노이즈가 많음. 핵심 키워드 2-4개.
- **언어 맞추기** — 한국어 정보는 한국어 쿼리, 영문 기술 문서는 영어 쿼리.
- **반복 금지** — 같은 의도로 쿼리 3회 이상 시도 시 전략 변경.
- **1차 쿼리 결과 평가 후** — 상위 결과의 키워드를 재활용해 2차 쿼리 개선.

## max_results 가이드

| 목적 | max_results |
|------|-------------|
| 빠른 사실 확인 | 3 |
| 일반 리서치 | 5 (기본값) |
| 다출처 검증 필요 | 8-10 |
| 종합 분석 | 10 |

## 결과 평가 기준

1. **날짜** — 최신 날짜 우선. 1년 이상 된 정보는 현재 유효 여부 확인.
2. **출처 신뢰도** — 공식 문서 > 전문 미디어 > 커뮤니티.
3. **관련성** — 제목/snippet이 쿼리 의도와 일치하는지.
