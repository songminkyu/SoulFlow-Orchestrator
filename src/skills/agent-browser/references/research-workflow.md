# 심층 리서치 워크플로우

## 단계별 상세 절차

### 1. Scope (범위 정의)

- 목표를 1문장으로 재진술
- 제약 추출: 기간, 지역, 언어, 출처 유형, 출력 형식
- 모호하면 가장 일반적인 해석으로 진행 후 불확실성 명시

### 2. Search (탐색)

- 첫 쿼리: 핵심 키워드 2-4개, 좁게
- 결과 평가 후 키워드 재조합해 2-3차 쿼리
- 한국어/영어 혼합 시 각각 별도 쿼리

### 3. Gather (수집)

- `web_fetch`로 상위 3-5개 URL 본문 추출
- 각 소스에서: 제목, URL, 날짜, 핵심 사실 1-2개만 추출
- JS/로그인 필요 시 `web_browser` 전환

### 4. Verify (검증)

| 검증 항목 | 기준 |
|----------|------|
| 날짜 | 1년 이상 된 정보는 현재 유효 여부 재확인 |
| 출처 충돌 | 2개 이상 충돌 시 Uncertainties에 명시 |
| 신뢰도 | 공식 문서 > 전문 미디어 > 커뮤니티 |

### 5. Synthesize (종합)

- 직접 답변 먼저, 배경 설명은 후순위
- 사실(확인됨)과 추론(추정됨)을 명확히 구분
- 날짜는 명시 (`2024-03 기준`)

### 6. Cite (출처)

- 주장 클러스터당 링크 1개
- 링크 없는 주장은 "출처 불명" 표기
- 링크 덤핑(모든 URL 나열) 금지

## 출처 신뢰도 계층

```
Tier 1 (최우선): 공식 문서, 정부 기관, 학술 논문
Tier 2:          주요 미디어, 기술 전문 블로그, GitHub 공식 저장소
Tier 3:          커뮤니티 포럼, 개인 블로그
Tier 4 (주의):   SNS, 익명 출처
```

## 한계 명시 기준

다음 중 하나라도 해당되면 Uncertainties에 반드시 기록:

- 정보 날짜가 6개월 이상 경과
- 출처 간 내용 충돌
- 직접 확인 불가능한 주장
- 단일 출처에만 의존
