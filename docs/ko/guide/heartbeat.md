# Heartbeat (주기 실행)

HeartbeatService는 설정된 주기마다 `workspace/HEARTBEAT.md`를 읽고, 내용이 있으면 에이전트에게 실행을 위임합니다.

## 동작 원리

```
타이머 틱
  → HEARTBEAT.md 읽기
  → 내용이 비어있으면 → 건너뜀
  → 내용이 있으면 → 에이전트 실행 ("HEARTBEAT.md를 읽고 지시를 따라라")
  → 에이전트가 HEARTBEAT_OK 반환 시 → 별도 알림 없음
  → 그 외 결과 → 채널로 보고
```

## 활성화 / 비활성화

`workspace/HEARTBEAT.md`의 내용 유무로 제어합니다.

**비활성 (기본)**: 주석, `#` 헤딩, `- [ ]` 체크박스만 있으면 실행하지 않습니다.

```markdown
# HEARTBEAT
<!-- 비어있으면 비활성 -->
```

**활성**: 주석이 아닌 실제 작업 내용을 추가합니다.

```markdown
# HEARTBEAT

디스크 사용량이 80% 이상이면 #alerts 채널에 알림을 보내줘.
```

## 작업 작성 가이드

구체적이고 조건이 명확한 지시가 가장 효과적입니다.

```markdown
# HEARTBEAT

## 시스템 모니터링
- 디스크 사용량이 80% 초과 시 즉시 Slack #alerts 채널에 알림
- 메모리 사용량이 90% 초과 시 즉시 알림

## 일일 리포트 (아침)
오전 9시 기준으로 오늘의 예약 작업 목록을 Telegram으로 요약해줘.
기준: KST 09:00 이후 첫 heartbeat 실행 시

## 서비스 점검
/doctor 결과에 이상 항목이 있으면 보고해줘.
```

## 자주 쓰는 패턴

### 조건부 알림

```
[대상 서비스]의 [지표]가 [임계값]을 초과하면 [채널]에 [형식]으로 알림을 보내줘.
```

### 주기 리포트

```
[주기 조건]에 [내용]을 [대상 채널]에 보내줘.
예: "매일 오전 첫 실행 시 전일 작업 요약을 Slack에"
```

### 상태 점검

```
[서비스/명령]을 실행하고 [조건]이면 [액션]을 취해줘.
```

## 대시보드에서 편집

대시보드 → **Templates** 페이지 → `HEARTBEAT` 항목에서 편집할 수 있습니다.
편집 내용은 `workspace/templates/HEARTBEAT.md`에 저장되며, 참조 템플릿입니다.
**실제 런타임 파일**은 `workspace/HEARTBEAT.md`입니다.

## 실행 주기 설정

실행 주기는 대시보드 → **Settings**에서 `heartbeat.interval_s` 값을 변경하여 조정합니다 (기본값: 300초).
